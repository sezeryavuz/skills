# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in our repository](https://github.com/changesets/changesets)

We have a quick list of common questions to get you started engaging with this project in
[our documentation](https://github.com/changesets/changesets/blob/main/docs/common-questions.md)

## How this repo uses changesets

This is a single-package repo of agent skills. Every change worth releasing gets a changeset:

1. Run `npx changeset` and follow the prompts. Pick the bump:
   - **patch** — a tweak to an existing skill (wording, a fixed reference, a no-op cut)
   - **minor** — a new skill, or a meaningful new capability in an existing one
   - **major** — a breaking change (a renamed/removed skill, a changed invocation contract)
2. Write the summary as a user-facing changelog entry — what changed and why it matters.
3. Commit the generated `.changeset/*.md` file alongside your code change.

On push to `main`, the release workflow opens (or updates) a "Version skills" PR. Merging that PR
runs `changeset version` to bump `package.json`, fold the changesets into `CHANGELOG.md`, and tag.
